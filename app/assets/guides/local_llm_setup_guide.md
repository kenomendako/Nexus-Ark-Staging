# 💻 Nexus Ark ローカルモデル導入ガイド

Nexus Arkでは、クラウドAPI（Gemini等）を使わず、ご自身のPCのグラフィックボード（GPU）を活用して完全オフラインでAIを稼働させる「ローカルモデル」に対応しています。

## 💡 どちらの方法を選べばいいですか？

Nexus Arkでは主に2つの導入方法があります。

1. **Ollama（オラマ）を使用する【推奨・初心者向け】**
   - **メリット**: インストールが非常に簡単。多くのモデルをコマンド一つで切り替え可能。
   - **デメリット**: Nexus Ark外部のツールをインストールする必要がある。

2. **GGUF直接ロード（llama-cpp）を使用する【上級者・高速化向け】**
   - **メリット**: Nexus Ark内で完結する。GPUの性能を最大限に引き出すための細かい調整が可能。
   - **デメリット**: 初回のエンジンインストール（コンパイル）に手間がかかる場合がある。

---

# 🦙 1. Ollama を使用した導入手順（推奨）

最も手軽にローカルモデルを動かせる方法です。

## 📥 ステップ 1: Ollama のインストール

1. **[Ollama 公式サイト](https://ollama.com/)** にアクセスし、OSに合ったインストーラーをダウンロード＆インストールします。
2. インストールが完了すると、自動的にバックグラウンドでOllamaが起動します（Windowsの場合はタスクトレイにアイコンが表示されます）。

## 🤖 ステップ 2: モデルのダウンロード

1. コマンドプロンプト（Windows）またはターミナルを開きます。
2. 以下のコマンドを入力して好みのモデルをダウンロードします。
   * (例: 軽量なQwen 2.5 3B の場合) `ollama run qwen2.5:3b`
   * (例: Gemma 2 2B の場合) `ollama run gemma2:2b`
3. モデルのダウンロード完了後、対話モードに入ったら `/bye` と入力して終了します（Ollama本体はバックグラウンドで動き続けます）。

## ⚙️ ステップ 3: UI からの自動設定と紐付け

1. **Nexus Ark を起動** します。
2. ブラウザでUI画面を開き、左側メニューの **「共通設定 (Global Settings)」** を開きます。
3. **「APIキー・モデル」** 内の **「💻 ローカルLLM (Ollama / GGUF)」** を開きます。
4. **「Ollama用の接続設定を追加」** ボタンをクリックします。これにより、接続情報が自動で登録されます。
5. 直後の「プロファイル名」で **「Ollama (Local)」** を選択します。
6. 「モデル名」欄に、ステップ2でダウンロードしたモデル名（例：`qwen2.5:3b`）を入力し、設定を保存します。

## 🗣️ ステップ 4: ルーム設定の変更

1. チャットしたいルームの設定を開きます。
2. **「APIプロバイダ」** で、**「OpenAI互換 (OpenRouter / Groq / Ollama / Zhipu AI)」** を選択します。
3. ルーム設定を保存し、チャットを開始してください！

# 🚀 2. GGUF直接ロード（llama-cpp）導入手順

GPU性能を極限まで引き出したい場合や、モデルファイルを直接管理したい場合の方法です。

## ⚠️ 注意: 自動インストールされない理由
このエンジン（`llama-cpp-python`）は、お使いの環境に合わせてその場でプログラムを最適化・コンパイルするため、手動でのセットアップが必要です。

## 🛠️ ステップ 1: 専用エンジンのインストール

### Windows の場合
1. Nexus Arkフォルダ内の `SetupLocalLLM.bat` をダブルクリックして実行してください。
2. 構築が完了するまで画面を閉じずにお待ちください。

### Linux / WSL2 環境の場合
ターミナルで以下のコマンドを実行してください：
```bash
CMAKE_ARGS="-DGGML_CUDA=on" uv pip install llama-cpp-python
```

## ⚙️ ステップ 2: モデルファイルの準備
1. HuggingFace等から **GGUF形式** のモデルをダウンロードします。
2. Nexus Arkフォルダ内の `models` フォルダなどに配置します。

## 🔗 ステップ 3: UI からの設定
1. **「共通設定」** > **「APIキー・モデル」** > **「Local Model Path (For GGUF)」** を探します。
2. 設置した GGUF ファイルの **フルパス** を入力し、保存します。

## 🗣️ ステップ 4: ルーム設定の変更
1. ルーム設定の **「APIプロバイダ」** で、**「ローカル (GGUF直接ロード)」** を選択します。
2. 初回の送信時はロードのため数分かかる場合があります。

---

## 💡 トラブルシューティング

### メモリ（VRAM）不足エラー
もしエラーが発生した場合は、一度黒いターミナル画面（コンソール）自体を完全に閉じてから、再起動してください。GPUメモリが解放されていない可能性があります。

### 回答が遅い
Nexus Arkは非常に長い記憶を送るため、処理に数分かかることがあります。裏側のコンソール画面で読み込み状況を確認しつつ、気長にお待ちください。
