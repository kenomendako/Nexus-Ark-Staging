# 🧪 Windows 実機テストガイド

Nexus Ark の配布パッケージが Windows 環境で正しく動作するか検証するための手順書です。
ステージングリポジトリ (`kenomendako/Nexus-Ark-Staging`) を使って、本番公開前にテストを行います。

## ⚠️ 重要：開始前のバックアップ
既存ユーザー（娘さん等）のPCでテストを行う際は、**作業前に必ず既存の `Nexus Ark` フォルダ全体を別の場所にコピー（バックアップ）してください。**
マイグレーション機能は安全に設計されていますが、万が一のデータ消失を防ぐためです。

---

## 🛠️ テスト環境の前提

1. **Windows PC**: テスト対象のマシン
2. **ステージングリポジトリ**: テスト対象のコードが `https://github.com/kenomendako/Nexus-Ark-Staging` の `main` ブランチにプッシュされていること
   - 開発環境で `ALLOW_STAGING_PUSH=true python tools/deploy_to_staging.py` を実行済み

---

## シナリオ A: 新規インストールテスト (ZIP版)

**目的**: 初めて使うユーザーが問題なく導入できるか確認する。

### 手順

1. **ZIPファイルの取得**:
   - ステージングリポジトリ `https://github.com/kenomendako/Nexus-Ark-Staging` にアクセス
   - 緑の `<> Code` ボタン → `Download ZIP` をクリック
   - もしくは、開発環境で `tools/zip_dist.py` で作成した ZIP を直接使用

2. **展開と起動**:
   - ダウンロードした ZIP を適当な場所に展開
   - `Start.bat` をダブルクリックで起動
   - **🔴 チェックポイント 1**: 「コマンドとして認識されません」エラーが出ないか？
     - 出た場合: `.bat` ファイルの改行コードが LF になっている → ビルドスクリプトを確認

3. **初期設定**:
   - WebUI が開き、APIキー設定画面（または初期設定ウィザード）が表示されるか
   - 設定完了後、メイン画面に遷移するか

4. **基本動作確認**:
   - ペルソナと会話できるか
   - 「今のバージョンは何？」と聞いて、正しく答えられるか
   - UIの各タブが正常に表示されるか

## シナリオ B: 自動更新テスト (tufup)

**目的**: 既存ユーザーの環境で自動更新が正しく動作するか確認する。

### 前提
- 旧バージョンの Nexus Ark が Windows にインストール済み（または旧バージョンの ZIP を展開済み）

### 手順

1. **update_url の変更**:
   - 旧バージョンの `config.json` を開く
   - `update_url` の値をステージングURLに変更:
     ```json
     "update_url": "https://raw.githubusercontent.com/kenomendako/Nexus-Ark-Staging/main/updates/"
     ```

2. **更新確認**:
   - Nexus Ark を起動
   - 設定 > 更新確認で新バージョンが検出されるか

3. **更新適用**:
   - 更新を適用
   - 自動再起動後、新バージョンで正常に起動するか
   - **🔴 チェックポイント**: バージョン表示が新しい番号になっているか

4. **データ整合性の確認**:
   - 会話ログが消えていないか
   - ペルソナの設定・記憶が保持されているか

## シナリオ C: マイグレーションテスト

**目的**: 旧バージョンからのデータ移行が正しく機能するか確認する。

### 手順

1. **準備**:
   - テスト対象PCに「旧バージョンの Nexus Ark フォルダ（バックアップ）」がある状態にする
   - ステージングリポジトリからクリーンインストール（シナリオA）の状態にする

2. **移行実行**:
   - 起動時に旧フォルダが同階層にあれば自動でウィザードが出る設計の場合、それを確認
   - 手動の場合は「設定」タブから「データ移行ツール」を使用

3. **確認**:
   - 旧環境の「会話ログ」が引き継がれているか
   - 「記憶」（Long-term memory）が反映されているか
   - ペルソナ設定が適切にマージされているか
   - **重要**: ユーザーの会話ログや設定が**消えていない**ことを確認

---

## テスト完了後の本番デプロイ

すべてのテスト項目がパスしたら、開発環境で以下を実行して本番公開します：

```bash
ALLOW_PUBLIC_PUSH=true python tools/promote_to_production.py
```
